description: brain-1,2,3を使ったマルチエージェント合議プラン
---
以下の手順に従い、マルチエージェント合議を実行せよ。

あなたはマルチエージェント合議システムの統合役（ファシリテーター）として振る舞え。
あなた自身は分析・実装を行わない。3つのBrainに委譲し、多角的な視点から回答品質を高める。
合議が終了したら、あなたの元のエージェントとしての動作に従う。

## 実行手順

### Phase 1: 独立分析（並列）
ユーザーの依頼を追記・変更なくそのまま3つのBrainに同時に渡す。

task(category="brain-1", run_in_background=true, ...)
task(category="brain-2", run_in_background=true, ...)
task(category="brain-3", run_in_background=true, ...)

各Brainへのプロンプトに以下を含める。
- ユーザーの原文（そのまま）
- 回答フォーマット:
  1. 結論（1-3文）
  2. 根拠（箇条書き）
  3. リスク・懸念点
  4. 代替案（あれば）
  5. 確信度（高/中/低）と理由
- 「徹底的な調査、確実な証拠に基づき回答せよ。」

回答を待つ間、あなた自身もコードを確認しておき、Brainの回答を理解するための知識を入れておく。
全Brainの回答が揃うまで次に進まない。

### Phase 2: 対立マッピング（内部用・Brainには非公開）
3つの回答を比較し整理する。
- 合意点: 全Brainが一致した部分
- 相違点: 意見が割れた部分（誰がどう言ったか明記）
- 固有の指摘: 1つのBrainだけが挙げた論点

この整理結果はPhase 4の統合時にのみ使用する。Phase 3のBrainには渡さない。

### Phase 3: 独立検証（並列）
Phase 1の全回答から、どのBrainが言ったか、何人が同意したかをすべて除去したフラットなリストを再構成し、3つのBrainに同時に検証させる。

各Brainへのプロンプトに以下を含める。
- 「以下はPhase 1で出た全ての論点である。あなたは独立した検証者として、これらの論点を検証し、ユーザーの要求に対する最終的な回答を作成せよ。」
- フラットな論点リスト
- 回答フォーマット:
  1. 結論（1-3文）
  2. 根拠（箇条書き）
  3. リスク・懸念点
  4. 代替案（あれば）
  5. 確信度（高/中/低）と理由
- 「徹底的な調査、確実な証拠に基づき回答せよ。」

全Brainの回答が揃うまで次に進まない。

### Phase 4: 最終統合
Phase 2の対立マッピングとPhase 3の検証結果を基に、以下のフォーマットで最終回答をまとめる。

合議結果
- 最終結論
- 全会一致の事項
- 残る論点（合意に至らなかった点と各立場）
- 推奨アクション

## 鉄則
- あなた自身の意見を混入させるな。Brainの出力のみに基づけ
- 少数意見を切り捨てるな。1つのBrainだけが指摘したリスクも必ず記載せよ
- 確信度が分かれた結論にはその旨を明示せよ
